Data Analysis with Obesity Dataset

Summary

Objective : Develop an advanced predictive model to accurately categorize individuals as "obese" or "non-obese" based on provided data.

Requirement : Develop a model that accurately classifies individuals as "obese" or "non-obese" based on characteristics such as height, weight, age, gender, BMI, and physical activity level.

Problem Type - CLASSIFICATION PROBLEM

MODEL Tested:

  1. Logistic Regression
  2. Linear Discriminant Analysis (LDA)
  3. Quadratic Discriminant Analysis (QDA)
  4. Decision Tree
  5. Random Forest

Techniques Used: One-Hot Encoding, SMOTE (Synthetic Minority Oversampling Technique) , PCA (Principal Component Analysis)

Implication: Achieving near-perfect accuracy (~100%) across models like decision trees, random forests, and logistic regression suggests potential issues, prompting a need for reverse engineering. This high accuracy may indicate data leakage (where the model relies on one or two features without learning broader patterns), overfitting (where the model memorizes instead of generalizing), or insufficient variability in the dataset.

Major Setback in Analysis: With limited records for obese individuals, categorizing all other groups—such as underweight, normal weight, and overweight—as "non-obese" has further imbalanced the dataset, resulting in data misalignment. Additionally, applying techniques like SMOTE to balance the obese and non-obese classes exacerbates this misalignment by disproportionately increasing the count of obese records across age, gender, and physical activity levels, leading to potential biases in the model.

Steps Followed:

  1. About Dataset - Understanding and considering different variables and checking the number of records recorded.
  2. EDA - The initial analysis involves examining the characteristics of each independent variable, assessing linearity, and evaluating data distribution. This includes checking for outliers, null values, and duplicate records to ensure data quality and reliability before model development. Result : No significant differences or anomalies were identified.
  3. DATA PRE-PROCESSING - One-hot encoding is employed to transform categorical variables, such as gender and obesity category, into numerical columns, facilitating their use as inputs for model building. NOTE: Since the requirement is more focused towards obese people, so OBESTIY_CATEGORY is divided into two categories only - Obese and Non-Obese. This created a huge imbalance in the dataset - (191 - Obese ; 809 - Non-Obese).
  4. MODEL BUILDING - Verifying the assumptions of the selected classification models and conducting an initial round of testing. Result: One major outcome is to remove the BMI column because 1) Since BMI is highly correlated with other variables (Weight, Height, Obesity_Category) 2) Data in BMI is not normally distributed 3) High Variance Inflation Factor (VIF) in BMI
  5. PRE-PROCESSING - An additional dataset was created excluding the BMI variable to assess the impact of its absence on the results.
  6. Model Building - Again tested both balanced and non-balanced dataset on dataset without BMI variable. Results: The model results revealed a decrease in accuracy and F1 score for logistic regression, decision tree, and random forest models upon excluding the BMI variable, highlighting its significance in the analysis. This is consistent with expectations, as BMI is derived from weight and height.
  7. EDA - Upon closer examination of both the balanced and imbalanced datasets, it was observed that SMOTE disproportionately increased the number of obese records relative to non-obese records, further skewing the dataset. For instance, the number of obese records surged beyond those of non-obese individuals in age categories such as 18-27, 28-37, and 68-77, where non-obese records were previously more prevalent. A similar trend was noted with gender, as the count of female records (581) rose significantly compared to the non-obese category (424).

Next Steps:

  1. Applying clustering techniques for groups like normal weight, underweight, overweight, and obese may yield better results, as the original dataset is proportionately distributed across these categories. This approach reduces the risk of misalignment since balancing the dataset would no longer be necessary.
  2. Since logistic regression assumes a sufficiently large dataset, collecting more data could enhance analysis accuracy and improve results.
  3. Hyperparameter tuning, such as using lbfgs as a solver for logistic regression instead of newton-cg, has shown to improve model performance. Further tuning could optimize results.
  4. Addressing assumptions for other models, such as applying dimensionality reduction to mitigate covariance, could improve compatibility with models like LDA, which assumes no multicollinearity.
  5. Including additional variables, such as sleep cycle, could enhance predictability in classifying obesity categories.
